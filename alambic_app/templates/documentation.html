Summary

ALAMBIC is a no-code Artificial Intelligence (AI) platform using an Active learning (AL) approach.

It allows for the development of a machine learning model with data partially (or even not) labelled where the model is able to communicate to the human which data point (or query) it would like to learn the true label of according to a specific strategy, such as choosing the query the most different from all the other queries or the query being the most representative of a population of queries. The model stops asking for true labels when reaching a criterion (also called the stopping criterion), like reaching a desired accuracy for the model performance or a specific number of annotated queries.

ALAMBIC also allows the study of the performance of different query strategies in an active learning setting for a specific model.

What can you do with ALAMBIC ?

ANNOTATE YOUR DATA

Annotations tools are available for diverse type of tasks and integrated in the active learning process.

TRAIN A MODEL WITH ACTIVE LEARNING

Develop a model with little to no data labelled and optimize the labelling of your data by having the model telling you which data point it is most interested to know the true label of. You have the control on the model, its settings, the features used, as well as the settings of the active learning.

EXPLORE THE PERFORMANCE OF THE FRAMEWORK

You can track either the performance of the model during its training or at the labelling step, or even study the impact of different active learning approaches for a specific model.

Active learning

With ALAMBIC, you can either develop a model with the help of active learning, meaning that you can also label in an optimized way (according to your chosen strategy) your data at the same time as you train a chosen model, or you can analyse the impact of different query strategies on the performance of a model.

Which one should I choose ?

If you are working with a partially labelled dataset and you want to label the rest, while still optimizing the amount of labels you have to provide, then developing a model is the task to choose. The obtained labels are enriching your dataset and the obtained model can label the rest of your data with a specified accuracy based on the given information.

If you want to study which query strategy would be more convenient for your dataset (prior to the task of developing a model, for example), or you want to compare the performance of a new query strategy your developed with other commonly used query strategies, then the analysis task is the way to go.


NOTE: In order to be performed successfully, the analysis must be performed with a dataset fully labelled. While we allow the annotation process to be done during the analysis, you would have to relabel for each repeat of the process the annotation.

Query strategies

Random sampling

Base method used to compare with the performance of other query strategies. It just selects the required number of queries in a random manner.

Uncertainty Sampling

The uncertainty sampling is the way the learner will select queries for whom it is the most uncertain in their label. This uncertainty can be measured in different ways. Let's define \(x\) as an instance, \(\hat{y_i}\) corresponds to the \(i\)th most likely class predicted for the instance \(x\).

Least confident

The simplest measure calculate the difference between 100% confidence and the most confident prediction obtained for the instance \(x\) $$LC(x) = 1 - P(\hat{y_1}|x)$$

Margin

Calculates the difference between the top two most confident predictions. $$M(x) = P(\hat{y_1}|x) - P(\hat{y_2}|x)$$ In that case, the strategy will select the instance with the smallest margin, since the smaller the margin is, the most unsure the decision.

Entropy

Calculates the difference between all the prediction, as defined by information theory. $$H(x) = -\sum_{k}P(y_k|x)\log(P(y_k|x))$$

Core-set

It is a diversity-based method which select in a greedy manner instances so that they are as different as possible from the training instances, but also as diverse as possible among each other. Based on the paper of Sener and Savarese (2018).

Stopping criteria

Stopping criteria are only used in the model development of the active learning framework. It will stop the active loop once a specific criterion has been reached.

You can find below the criteria currently available.

BUDGET

You can choose the amount of data labeled by the oracle

ACCURACY

The active loop will stop once the minimum accuracy desired is reached.
